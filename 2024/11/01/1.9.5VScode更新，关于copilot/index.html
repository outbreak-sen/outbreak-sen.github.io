1.9.5VScode更新，关于copilot

文档：https://code.visualstudio.com/docs/copilot/copilot-edits

我的copilot设置界面：https://github.com/github-copilot/signup/success?utm_campaign=2024q3-em-MSFT-signup&utm_medium=first&utm_source=vscode-completions-readme

如何打开

首先Setting: setting(github.copilot.chat.edits.enabled)

打开： Open Copilot Edits from the Chat menu, or press Ctrl+Shift+Alt+I.
将相关文件添加到工作集中，以向Copilot指示要处理的文件。

输入提示，告诉Copilot您要进行的编辑！例如，为所有页面添加一个简单的导航栏，或者使用vitest而不是jest。
如果您之前安装了GitHub Copilot，则会在您之前聊天的位置显示一个视图，使您能够将聊天视图还原到旧位置，如果这对您更有效的话。

现在可以进行代码审查，看看有没有错误的，风格对不对

当使用自定义查看说明查看选择时，您可以通过设置（github.copilot.chat.reviewSelection.instructions）来定义这些特定要求。与代码生成和测试生成指令类似

然后code-style可以这么写

内置代码编辑者

Setting: chat.experimental.detectParticipant.enabled

GitHub Copilot有几个内置的聊天参与者，比如@workspace。
为了使使用自然语言的聊天参与者更容易，Copilot chat将在可能的情况下自动将您的问题路由到合适的参与者。
输入聊天提示时的默认操作是发送和分派，其中分派包括参与者检测。如果您选择发送，请求将直接发送到Copilot Chat，不会自动发送给聊天参与者。

控制当前编辑器上下文

Copilot Chat始终会自动将您当前的选择或当前可见的代码作为上下文包含在您的聊天请求中。大型语言模型（LLMs）通常擅长理解一段上下文是否相关。但有时，当你问一个与你当前的编辑器无关的问题时，包括这个上下文可能会影响模型如何解释你的问题。
我们现在在聊天输入中显示一个特殊的附件控件，它给出了关于编辑器上下文的提示，并使您能够切换是否包含编辑器上下文。

符号连接

交互式工作区符号链接
Copilot Chat的一个常见用例是询问有关工作区中代码的问题，例如使用/tests为所选代码生成新的单元测试，或要求@workspace在项目中找到一些特定的类或函数。在这个里程碑中，我们为Copilot在聊天回复中提到的任何工作区符号添加了增强的链接。这些符号链接可以帮助您更好地理解副驾驶的反应，并了解其中使用的符号的更多信息。
符号链接在响应中呈现为小球，就像我们上一个里程碑添加的文件链接一样。要开始了解有关符号的更多信息，只需选择符号链接即可跳转到该符号的定义：
基本符号链接应该适用于支持Go to Definition的任何语言。更高级的IntelliSense选项，如Go to Implementations，也需要对该语言的支持。确保安装语言扩展，以获得Copilot响应中使用的任何编程语言的最佳符号支持。

问workspace问题

@workspace允许您询问有关当前项目中代码的问题。这是通过GitHub的代码搜索或VS code构建的智能本地索引来实现的。

如果需要对当前文件夹进行细致提问，那么copilot需要建立一个本地索引，这需要一段时间

聊天跟进改进

Setting: setting(github.copilot.chat.followUps)
为了在聊天视图中为聊天对话腾出更多空间，我们使后续提示更加简洁，默认情况下，它们只出现在第一个回合。配置设置（github.copilot.chat.followUps）设置：

  • firstOnly (default) - Follow-up prompts only appear on the first turn后续提示仅在第一回合出现
  • always - Follow-up prompts always appear始终显示后续提示
  • never - Disable follow-up prompts禁用后续提示

有几个扩展很有用

Web Search for Copilot

这个可以在Copilot中从网络上获取最新和最相关的信息。这由两个不同的搜索引擎之一提供支持，通过设置（websearch.preferencedEngine）

MermAId diagram generation with Copilot

为GitHub copilot贡献了一个新的聊天参与者，可以使用受Markdown启发的诊断和图表工具mermAId为您的代码构建和修改可视化。

  1. 在copilot中@MermAld创建一个代码图表

  2. 并使用/interate slash命令来优化图表。Slash命令可用于特定的图表，为模型提供额外的指导。

  3. 并且图表具有跳转功能

  4. 并且还有Visual Outline辅助，

  5. Mermaid Visual Outline view打开Visual Outline视图，从活动编辑器动态生成图表。你可以开始聊天，以便更好地控制。

Python/Pylance也更新了

  1. 生成注释：使用Pylance的docstring模板生成功能更方便地为Python代码生成文档！您可以通过键入“”或“”、按Ctrl+Space或选择灯泡来调用generate docstring代码操作，为类或方法生成docstring模板。生成的docstring包括函数描述、参数、参数类型、返回和返回类型的字段。

    setting python.analysis.supportDocstringTemplate:true）

  2. 折叠所有文档注释的字符串
    在顶部输入Pylance:Fold All docstrings就可以折叠所有字符串
